## Phase 1 — Critical vintage-correctness

### Task 1: C-13 — benchmark re-append idempotency

**Files:**
- Modify: `src/bls_stats/vintage/ledger.py` (`prior_benchmark_count` gains a `before_release` filter)
- Modify: `src/bls_stats/pipeline.py` (the `expand` callback passes the release date)
- Test: `tests/test_pipeline.py`

**Interfaces:**
- Consumes: `Ledger.prior_benchmark_count(program, ref_date)` (current signature), `expand(release, prior_benchmark)`.
- Produces: `Ledger.prior_benchmark_count(self, program: str, ref_date: date | None, before_release: date | None = None) -> int` — when `before_release` is given, only ingested rows with `release_date < before_release` count. The pipeline's `expand` callback passes `before_release=release.release_date`, so a benchmark window's counter is derived from prints at *strictly earlier* release dates and a re-poll of the same release is a no-op.

**Root cause (from audit C-13):** the window-slot benchmark counter is `prior_benchmark_count(...) + 1`, computed from already-ingested rows *including the rows this same release wrote on a prior run*. Re-polling the same benchmark release (feeds retain ~12 entries, ~a year) recomputes the counter as 2, 3, 4… — the idempotency guards `slot_status`/`slot_exists` key on that changing counter, so they never match and the whole window re-appends every run.

        )

### Task 2: C-14 — deny back-dated releases in `ingest` (catch-up must not fabricate prints)

**Files:**
- Modify: `src/bls_stats/pipeline.py` (`run_ingest` filters back-dated releases before processing)
- Test: `tests/test_pipeline.py`

**Interfaces:**
- Consumes: `run_ingest(...)`, `Ledger.resolved()`, `expand(...)`, `SlotRecord`.
- Produces: no signature change. New internal rule: a release is **back-dated** if a strictly-newer release for the same program exists either in the current poll batch or already ingested in the ledger. Back-dated releases are not fetched; their expanded routine slots are recorded `missed`, so `gaps` can surface them and no revised-data-as-old-print vintage is written.

**Root cause (from audit C-14):** on bootstrap-via-`ingest` or outage catch-up, `poll` returns several releases; `_fetch_event` always downloads the *current* LABSTAT file, so an older release gets stamped with its old `release_date` but carries today's revised values — the clairvoyance the vintage model exists to prevent. `backfill` is the intended catch-up tool.

```python
def _comparator(ledger: Ledger, program: str, revision: int | None) -> int | None:
    """ARCH §7.3: most recent ingested row_count for the same program and slot type.

    Prefers a same-revision comparator; falls back to the latest ingested row for the program
    regardless of revision (e.g. a backfill baseline with null revision) so the first live
    increment after a backfill is still row-band-checked rather than ungated (C-16)."""
    ingested = ledger.resolved().filter(
        (pl.col("program") == program) & (pl.col("status") == "ingested")
    )
    same = ingested.filter(pl.col("revision").eq_missing(pl.lit(revision, dtype=pl.Int16))).sort(
        "ingested_at", descending=True
    )
    if same.height:
        return int(same["row_count"][0])
    any_row = ingested.sort("ingested_at", descending=True)
    return int(any_row["row_count"][0]) if any_row.height else None
```

```python
@app.command()
def gaps(
    program: str | None = typer.Option(None),
    strict: bool = typer.Option(False, "--strict", help="missed prints also exit non-zero"),
    as_of_date: str | None = typer.Option(
        None, "--as-of-date", help="audit expected releases published on/before this YYYY-MM-DD"
    ),
) -> None:
    """Audit expected releases (from the calendar) against the ledger (ARCH §5.3, §8).

    A calendar row with a non-null `release_date` at or before the reference date is an
    *expected* release. If the ledger has no row for its `(program, ref_date)` (of any status),
    it is *unexplained* — a release the pipeline never recorded. Recorded `missed`/`deferred`
    slots are *acknowledged*: printed but not failing, unless `--strict` (which also fails on
    `missed`). The reference date defaults to the latest published `release_date` in the
    calendar; `--as-of-date` overrides it. Exits `1` on any unexplained gap, on a `--strict`
    missed slot, or if the calendar is unbuilt; `0` otherwise.
    """
    import polars as pl

    from bls_stats.vintage.ledger import Ledger

    _, store = _setup()
    cal = store.read_state("release_calendar")
    if cal is None:
        typer.echo("no calendar — run `bls-stats calendar build`", err=True)
        raise typer.Exit(1)
    ledger = Ledger(store).resolved()
    if program:
        cal = cal.filter(pl.col("program") == program)
        ledger = ledger.filter(pl.col("program") == program)
    published = cal.filter(pl.col("release_date").is_not_null())
    ref = (
        date.fromisoformat(as_of_date)
        if as_of_date
        else (published["release_date"].max() if published.height else None)
    )
    expected = (
        published.filter(pl.col("release_date") <= ref).select("program", "ref_date").unique()
        if ref is not None
        else published.select("program", "ref_date").unique()
    )
    recorded = ledger.select("program", "ref_date").unique()
    unexplained = expected.join(recorded, on=["program", "ref_date"], how="anti")
    acknowledged = ledger.filter(pl.col("status").is_in(["missed", "deferred"]))
    typer.echo(f"unexplained: {unexplained.height}  acknowledged: {acknowledged.height}")
    if unexplained.height:
        typer.echo(str(unexplained.sort("program", "ref_date")))
    missed = acknowledged.filter(pl.col("status") == "missed")
    raise typer.Exit(1 if (unexplained.height or (strict and missed.height)) else 0)
```

def _read_zip_csv(zip_path: Path) -> pl.DataFrame:
    """Read the single CSV member of a QCEW ZIP with locked dtypes, streaming from disk.

    Code columns are read as `Utf8` to preserve leading zeros and alpha `area_fips` values.
    The member is extracted to a temp file and scanned lazily with a streaming collect so peak
    memory stays bounded on multi-GB years (ARCH §10 < 8 GB target, C-19) — never decompressed
    whole into memory."""
    import tempfile

    schema = (
        {c: pl.Utf8 for c in _CODE_COLS}
        | {"year": pl.Int32, "qtr": pl.Int8}
        | {c: pl.Float64 for c in _VALUE_COLS}
    )
    with zipfile.ZipFile(zip_path) as zf:
        member = next(n for n in zf.namelist() if n.endswith(".csv"))
        with tempfile.TemporaryDirectory() as td:
            csv_path = Path(zf.extract(member, td))
            return pl.scan_csv(csv_path, schema_overrides=schema).collect(streaming=True)
